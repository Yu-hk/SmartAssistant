## 1. 系统概述

### 1.1 系统架构

SmartAssistant 智能客服系统采用多 Agent 协作架构，订单与退款相关操作由 **Order Agent（订单客服智能体）** 统一处理。

```
用户咨询 → 智能路由 → Order Agent
                          ├── queryOrder(orderId)    ← 订单查询
                          ├── applyRefund(orderId, reason)  ← 退款申请
                          └── trackLogistics(trackingNo)  ← 物流跟踪
```

### 1.2 适用范围

本手册适用于以下场景：

| 场景 | 处理方式 | 响应时间 |
|------|---------|---------|
| 订单状态查询 | 自动 | 即时 |
| 物流轨迹查询 | 自动 | 即时 |
| 退款申请（7天无理由） | 自动受理 | 即时 |
| 退款申请（质量问题） | 自动受理 + 人工审核标记 | 即时受理，24h 内审核 |
| 仅退款/退货退款 | 自动区分 | 即时 |
| 退款进度查询 | 自动 | 即时 |
| 异常订单（缺货/错发/漏发） | 自动受理 + 升级人工 | 即时受理，2h 内响应 |

### 1.3 系统入口

- **用户端**：APP/Web 在线客服入口、微信小程序
- **客服端**：SmartAssistant 管理后台 → 订单客服工作台
- **API 接口**：`POST /api/chat`，传入 `{"message": "用户问题", "sessionId": "会话ID"}`

## 3. 订单状态说明

### 3.1 状态流转图

```
待付款 ──→ 待发货 ──→ 已发货 ──→ 已签收
  │          │          │
  │          ├──→ 退款中 ──→ 已退款
  │          │
  └──→ 已取消
```

### 3.2 各状态说明

| 状态 | 定义 | 可操作 | 时效 |
|------|------|--------|------|
| **待付款** | 订单已创建，等待用户支付 | 取消订单、修改地址 | 24h 未支付自动取消 |
| **待发货** | 已支付，仓库准备中 | 申请退款、修改地址 | 48h 内发货（现货） |
| **已发货** | 已出库，物流途中 | 申请退款（需拒收）、物流查询 | 见物流信息 |
| **已签收** | 用户已确认收货 | 申请退货退款（7天内） | 签收后 7 天内 |
| **退款中** | 退款申请已受理 | 取消退款申请 | 见 §4.4 |
| **已退款** | 款项已原路返回 | — | 见 §4.4 |
| **已取消** | 订单已取消 | 重新下单 | — |

### 3.3 特别说明

> **已发货状态下申请退款**：需告知用户拒收包裹或在物流显示「派送中」时联系快递员退回。退款将在仓库收到退回商品后处理。

> **部分发货**：多件商品订单可能处于「部分发货」状态（如 iPhone 已发、AirPods 未发）。系统查询时会分别显示各商品状态。

## 4. 退款流程

### 4.1 退款类型划分

| 类型 | 适用场景 | 运费承担 | 退款金额 |
|------|---------|---------|---------|
| **仅退款** | 未发货、虚拟商品、缺货 | — | 全额 |
| **退货退款** | 7 天无理由 | 买家承担 | 商品金额 |
| **质量问题退款** | 功能故障、描述不符 | 卖家承担 | 全额（含运费） |
| **部分退款** | 少发配件、轻微瑕疵 | 卖家承担 | 协商金额 |

### 4.2 退款流程标准

```
用户发起退款申请
    │
    ├── 系统自动判断退款类型
    │     ├── 状态 = 待发货 → 仅退款
    │     ├── 状态 = 已发货（未签收）→ 仅退款（需拒收）
    │     └── 状态 = 已签收（7天内）→ 退货退款
    │
    ├── 系统执行：
    │     ├── 记录退款原因
    │     ├── 计算退款金额
    │     ├── 通知仓库/物流（如退货）
    │     └── 返回退款受理确认
    │
    └── 退款到账（按支付方式）
          ├── 支付宝/微信：1-3 个工作日
          ├── 银行卡：3-5 个工作日
          └── 信用卡：5-7 个工作日
```

### 4.3 退款申请参数说明

| 参数 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 订单号 | 是 | 需退款的订单编号 | `ORD-2024001` |
| 退款原因 | 是 | 简要说明 | `质量问题`、`不想要了` |
| 退款金额 | 否 | 具体金额（支持部分退款） | `3999.00` |
| 退货物流 | 否 | 退货后补充 | `中通 ZT123456` |

### 4.4 退款时效标准

| 退款类型 | 审核时效 | 到账时效 |
|---------|---------|---------|
| 仅退款（待发货） | 自动审核 | 1-3 个工作日 |
| 仅退款（已发货拒收） | 仓库确认收货后 | 仓库收到后 1-3 个工作日 |
| 退货退款（7天无理由） | 卖家 48h 内审核 | 卖家确认后 1-3 个工作日 |
| 质量退款 | 24h 内人工审核 | 审核通过后 1-3 个工作日 |

> **法律依据**：《中华人民共和国消费者权益保护法》第二十四条规定，经营者提供的商品或者服务不符合质量要求的，消费者可以依照国家规定、当事人约定退货，或者要求经营者履行更换、修理等义务。依照前款规定进行退货、更换、修理的，经营者应当承担运输等必要费用。

### 4.5 退款话术模板

| 场景 | 模板 |
|------|------|
| 仅退款受理 | "已为您提交订单 XXX 的退款申请。退款金额 XX 元将按原支付方式原路返回，预计 1-3 个工作日到账。" |
| 退货退款受理 | "已为您提交退货退款申请。请将商品寄回以下地址：XXX。退货完成后，退款将在 1-3 个工作日内到账。" |
| 退款进度查询 | "您订单 XXX 的退款当前状态为「退款中」，预计 XX 月 XX 日前到账。" |

## 6. 异常处理

### 6.1 异常场景分类

| 编号 | 异常类型 | 处理方式 |
|------|---------|---------|
| E-001 | 订单不存在 | 返回提示信息，建议核对订单号 |
| E-002 | 退款金额异常 | 提示联系人工客服 |
| E-003 | 重复退款 | 检测到已有退款中的记录，提示用户 |
| E-004 | 超出退换货期限 | 告知已超出 7 天无理由退货时效，引导联系人工 |
| E-005 | 物流单号无效 | 检查单号格式，提示重新输入 |

### 6.2 话术模板

**E-001 订单不存在**：
> "未找到订单 XXX，请确认订单号是否正确。订单号格式为 ORD- 开头的 10 位编号。"

**E-004 超出退换货期限**：
> "您订单 XXX 的签收时间为 XX 月 XX 日，已超出 7 天无理由退货时效（截至 XX 月 XX 日）。如商品存在质量问题，请联系人工客服为您处理。"

**E-005 物流单号无效**：
> "快递单号格式不正确，请确认后重新输入。一般的快递单号由数字和字母组成，长度为 10-15 位。"
