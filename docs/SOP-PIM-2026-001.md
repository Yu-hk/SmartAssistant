## 1. 系统概述

### 1.1 系统架构

SmartAssistant 智能客服系统中，商品信息由 **Product Agent（商品咨询智能体）** 统一处理。

```
用户咨询 → 智能路由 → Product Agent
                          ├── queryProductInfo(productCode)  ← 商品详情
                          ├── checkStock(productCode)        ← 库存查询
                          └── getPrice(productCode)          ← 价格查询
```

### 1.2 适用范围

| 场景 | 处理方式 | 响应时间 |
|------|---------|---------|
| 商品基本信息查询 | 自动 | 即时 |
| 商品价格查询 | 自动 | 即时 |
| 库存状态查询 | 自动 | 即时 |
| 商品规格/参数查询 | 自动 | 即时 |
| 商品对比 | 自动（同一品类） | 即时 |
| 分期信息查询 | 自动 | 即时 |

### 1.3 信息更新机制

商品信息通过以下渠道同步：

| 渠道 | 更新频率 | 优先级 |
|------|---------|--------|
| 商品管理系统（PMS） | 实时 | 最高 |
| 供应商数据推送 | 每日 | 高 |
| 运营后台手动维护 | 按需 | 中 |
| 价格监控系统 | 每 30 分钟 | 高 |

## 2. 商品信息查询

### 2.1 查询方式

| 触发方式 | 示例 |
|---------|------|
| 按商品编码查询 | `IPHONE-15-PRO`、`MACBOOK-AIR-M3` |
| 按商品名称查询 | `iPhone 15 Pro`、`MacBook Air` |
| 按关键词查询 | `苹果手机`、`笔记本`（模糊匹配） |

### 2.2 查询返回字段

```json
{
  "productCode": "IPHONE-15-PRO",
  "productName": "iPhone 15 Pro",
  "price": "8999.00",
  "stock": "充足",
  "spec": "A17 Pro芯片、4800万像素、钛金属边框",
  "colors": "原色钛金属/蓝色钛金属/白色钛金属/黑色钛金属"
}
```

### 2.3 查询失败处理

| 场景 | 系统回复 | 客服操作 |
|------|---------|---------|
| 商品编码不存在 | "未找到商品 XXX，请确认商品编码或名称是否正确" | 引导用户提供准确编码或商品名 |
| 模糊匹配多个结果 | 返回匹配度最高的结果 | 如不匹配，提示用户提供更精确信息 |
| 商品已下架 | "该商品已下架，建议查看同系列其他商品" | 推荐替代商品 |

## 3. 库存状态说明

### 3.1 库存状态定义

| 状态 | 含义 | 预计发货时间 | 客服回复 |
|------|------|-------------|---------|
| **充足** | 库存 ≥ 50 件 | 24 小时内发货 | "库存充足，下单后 24 小时内发货。" |
| **紧张** | 库存 5-49 件 | 3-5 天发货 | "库存紧张，建议尽快下单，预计 3-5 天发货。" |
| **缺货** | 库存 = 0 | 补货时间待定 | "暂时缺货，补货时间待定。需要我帮您到货通知吗？" |
| **预售** | 开放预订 | 按预售页面说明 | "该商品为预售商品，预计 XX 月 XX 日起按付款顺序发货。" |

### 3.2 库存分级管理

库存状态根据以下维度综合判定：

| 商品类型 | 安全库存线 | 预警库存线 | 补货周期 |
|---------|-----------|-----------|---------|
| 爆款商品（日均销量 > 100） | 500 | 200 | 每日 |
| 常规商品（日均销量 10-100） | 200 | 50 | 每周 |
| 长尾商品（日均销量 < 10） | 50 | 10 | 每两周 |
| 预售商品 | — | — | 按预售计划 |

### 3.3 库存信息特别说明

> **大促期间（618、双11、年货节等）**：库存数据以活动页面公示为准，系统库存数据可能存在延迟。建议客服引导用户以订单提交成功为准。

> **多仓库存**：部分商品支持多地发货，系统显示的库存为可售总库存。具体发货仓库由系统根据用户地址自动分配。

## 6. 常见问题

### 6.1 商品咨询高频问题

**Q: 商品是否正品？**
A: 平台所有商品均为正品行货，支持官方验货。部分品牌提供官方保修，具体以商品详情页说明为准。

**Q: 支持 7 天无理由退货吗？**
A: 本商品支持 7 天无理由退货（需商品完好、配件齐全、不影响二次销售）。定制类、鲜活易腐类商品除外。

**Q: 保修政策是怎样的？**
A: 商品保修政策以品牌官方规定为准。通常电子产品提供 1 年官方保修。建议保留购买凭证和包装。

**Q: 什么时候发货？**
A: 库存充足的商品下单后 24 小时内发货。库存紧张的商品预计 3-5 天发货。具体发货时间以下单后页面提示为准。

**Q: 可以开发票吗？**
A: 支持开具电子发票（增值税普通发票），下单时选择「需要发票」即可。发票将在发货后 7 个工作日内推送至您的账户。

### 6.2 拒赔与争议处理话术

**场景：用户质疑库存信息不准确**
> "非常抱歉给您带来困扰。库存数据实时变动，您看到的结果可能与实际库存存在短暂延迟。建议以下单成功为准，如订单因缺货无法发货，我们将第一时间联系您处理。"

**场景：用户要求价格保护**
> "感谢您的反馈。该商品当前价格与您购买时存在差异，我们很乐意为您处理价保。根据价保政策，签收后 7 天内降价可申请退还差价。请问您的订单号是？"

### 8.3 商品信息质量规范

| 检查项 | 标准 | 异常处理 |
|-------|------|---------|
| 商品名称 | 品牌 + 型号 + 规格，不超过 50 字 | 名称缺失时返回「未命名商品」 |
| 价格 | 必须为正数，保留两位小数 | 价格为 0 或负数时标记为「价格待确认」 |
| 库存 | 必须为非负整数 | 库存为空时标记为「请联系客服」 |
| 规格描述 | 包含核心参数（芯片、屏幕、存储等） | 描述缺失时标注「规格信息更新中」 |
| 颜色 | 品牌官方色号名称 | 颜色缺失时标注「颜色以实物为准」 |
