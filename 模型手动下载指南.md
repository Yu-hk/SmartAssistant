## 1. BGE-large-zh-v1.5 ONNX 模型（1.2 GB）

### 方案一：使用 Python 下载并转换（推荐）

> 在你的**个人电脑**（非本环境）上执行，网络更快、环境更稳定。

#### 步骤 1：安装依赖
```bash
pip install huggingface-hub torch transformers onnx onnxruntime
```

#### 步骤 2：下载 PyTorch 模型
```bash
# 从 HuggingFace 镜像下载（国内更快）
export HF_ENDPOINT=https://hf-mirror.com

# 或者用 huggingface-cli
huggingface-cli download BAAI/bge-large-zh-v1.5 \
  --local-dir ./bge-large-zh-v1.5 \
  --local-dir-use-symlinks False
```

#### 步骤 3：转换为 ONNX
运行项目中的转换脚本：

```bash
python convert_bge_to_onnx.py \
  --input ./bge-large-zh-v1.5 \
  --output ./bge-large-zh-v1.5.onnx
```

> ⚠️ 这个脚本会自动包含 `token_type_ids` 输入，与 Java 代码兼容。

#### 步骤 4：复制到项目目录
将生成的 `bge-large-zh-v1.5.onnx` 和 `bge-large-zh-v1.5.onnx.data` 复制到：

```
D:\workspace\SmartAssistant\models\
├── bge-large-zh-v1.5.onnx        # ← 放这里
├── bge-large-zh-v1.5.onnx.data   # ← 放这里
├── bge-small-zh-v1.5.onnx        # ✅ 已有
├── bge-small-zh-v1.5.onnx.data   # ✅ 已有
└── tokenizer.json                # ✅ 已有
```

## 2. Ollama + qwen2.5:7b（4.5 GB）

### 步骤 1：下载 Ollama

访问 [https://ollama.com/download](https://ollama.com/download) 下载 Windows 安装包。

或者用你的包管理器：
```bash
# Windows (winget)
winget install Ollama.Ollama

# macOS

# Linux

```

安装后，Ollama 通常会自动启动为后台服务。你也可以手动启动：
```bash
ollama serve
```

## 4. 常见问题

### Q: 什么是 `.onnx.data` 文件？
这是 PyTorch 2.x 新版本导出 ONNX 时额外生成的外部权重文件。ONNX 运行时加载 `.onnx` 文件时会自动在同目录查找 `.onnx.data`。

### Q: BGE-small 和 BGE-large 有什么区别？
| 模型 | 向量维度 | 大小 | 用途 |
|------|---------|------|------|
| `bge-small-zh-v1.5` | 512 | 92 MB | 路由缓存语义匹配（轻量） |
| `bge-large-zh-v1.5` | 1024 | 1.2 GB | RAG 检索（高质量） |

### Q: 没有 BGE-large 模型时项目能运行吗？
可以，但部分功能会降级：
- **普通聊天** ✅ 正常工作（不依赖 BGE 模型）
- **Router 语义缓存** ⚠️ 会 fallback 使用 BGE-small 模型（已配置好）
- **RAG 检索** ❌ 无法执行（需要 BGE-large）
- **向量搜索** ❌ 无法执行（需要 BGE-large）

### Q: 没有 Ollama 时项目能运行吗？
可以编译，但所有 LLM 调用（聊天、SQL 生成等）都会失败，因为 LLM 依赖 Ollama API (`http://localhost:11434`)。
